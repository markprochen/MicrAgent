# 🚀 Agent 技能索引清单 (Manifest)

本系统采用模块化分级加载架构。以下是当前已激活的技能包及其职责概述。如果你需要调用具体函数但不知道参数格式，请务必先使用 `get_skill_detail(skill_name)` 获取对应的说明手册。

## 1. 核心系统工具 (until)
- **模块名称**: `until`
- **核心职责**: 负责系统环境感知与基础文件管理。
- **关键能力**: 获取当前日期、查询用户画像/背景记忆、读写本地纯文本文件、创建目录、给文件添加文本信息以及递归扫描项目文件结构。

## 2. Excel 数据中枢 (excel_utils)
- **模块名称**: `excel_utils`
- **核心职责**: 专门处理 `.xlsx` 格式的表格数据。
- **关键能力**: 初始化带表头的 Excel 文件、读取表格内容（转为 JSON 列表）、向现有表格追加数据、获取表格元数据（工作表和列名）以及合并多个 Excel 文件。

## 3. 互联网搜索引擎 (web_search)
- **模块名称**: `web_search`
- **核心职责**: 获取实时外部信息。
- **关键能力**: 使用 DuckDuckGo 搜索引擎获取最新的新闻、技术文档或本地知识库中不存在的事实。

## 4. 网页深度交互 (web_browser)
- **模块名称**: `web_browser`
- **核心职责**: 模拟真实浏览器行为。
- **关键能力**: 访问指定 URL 并提取网页正文内容、在网页上执行点击操作、在输入框中录入文本。

## 5. 自我学习能力
- **模块名称**: `self_learning`
- **核心职责**: 创建自己的skill行为 使用 `write_python_skill`函数创建你的logic.py
- **关键能力**: 核心自我进化模块。定义了如何创建新技能、如何编写技能代码及文档的标准化流程。当需要扩展新功能或学习新工具时必读。
- **JSON 规范**: 在 Action Input 中，必须使用双引号 `"`。如果内容包含 Python 代码，请务必将代码中的引号转义。
- **原子化操作**: 不要在一个 Action 中写入过长的代码。先写 `__init__.py`，再写 `logic.py`，最后写 `doc.md`。
- **验证流程**: 写入文件后，你可以直接调用 `get_skill_detail`。系统会在你写入后自动刷新技能库。
---

### 💡 调度逻辑指引
1. **优先检索**: 处理任务前，先通过 `until` 模块确认日期或用户偏好。
2. **按需加载**: 识别到任务属于上述某个分类时，调用 `get_skill_detail` 加载对应的 `doc.md` 以获取 `Action Input` 的精确 Schema。
3. **组合规划**: 涉及复杂任务（如：搜索并存入 Excel）时，应跨模块组合调用。
